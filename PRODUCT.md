# Product

## Register

brand

## Users

Repo-Docs is for developers, maintainers, and coding-agent users who are building or reviewing repositories that change quickly through AI-assisted coding. They arrive at the homepage to understand what the skill does, inspect real generated files, and decide whether it is worth installing.

## Product Purpose

Repo-Docs keeps repository understanding close to the source by generating walkthroughs, concept pages, references, glossaries, change logs, and sync rules. The homepage succeeds when a visitor can quickly grasp the skill's evidence-first promise, see complete generated examples from real non-skill GitHub projects, and copy or follow the install path.

## Brand Personality

Refined, scholarly, and evidence-obsessed. The page should read like a carefully edited evidence atlas for serious repository work: calm typography, clear hierarchy, and source-grounded examples. Atmosphere comes from restraint and artifact detail, not game UI or loud decorative effects.

## Anti-references

Avoid generic SaaS dashboards, plain README-like pages, AI-looking fantasy art, loud pixel-game UI, decorative terminal panels with no substance, fake demo cards, and project cases that only show repository names instead of generated file contents.

## Design Principles

- Show the artifact: real generated Markdown files should be visible and inspectable.
- Make the metaphor useful: atlas, route, inventory, and quest language should clarify repo understanding rather than becoming decoration.
- Keep evidence in view: claims should point toward source-grounded outputs, file trees, and verification.
- Reward exploration: browsing cases and modes should feel tactile without hiding information.
- Stay installable: the visual richness should never bury the quick-start path.

## Accessibility & Inclusion

Target WCAG AA contrast for text, visible keyboard focus, usable file-tree controls, reduced-motion alternatives, and responsive layouts that remain readable on mobile and wide desktop screens.
